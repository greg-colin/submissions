Method Detail
<inner> init()
This function does some initial setup that should only occur once, particularly setting the lastTime variable that is required for the game loop.
<inner> main()
The main loop of the game engine. This function serves as the kickoff point for the game loop itself and handles properly calling the update and render methods.

The loop sets a variable to the length of time between calls to win.requestAnimation frame. This value is later passed to each GameCharacter object's update method.

See:
Player#update
Enemy#update
<inner> render()
This function initially draws the "game level", it will then call the renderEntities function. Remember, this function is called every game tick (or loop of the game engine) because that's how games work - they are flipbooks creating the illusion of animation but in reality they are just drawing the entire screen over and over.
<inner> renderEntities()
This function is called by the render function and is called on each game tick. It's purpose is to redraw the background and score, then call the render functions you have defined on your character entities within app.js

This function does nothing but it could have been a good place to handle game reset states - maybe a new game menu or a game over screen those sorts of things. It's only called once by the init() method.
<inner> update(dt)
This function is called by main (our game loop) and itself calls all of the functions which may need to update entity's data. Based on how you implement your collision detection (when two entities occupy the same space, for instance when your character should die), you may find the need to add an additional function call here. For now, we've left it commented out - you may or may not want to implement this functionality this way (you could just implement collision detection on the entities themselves within your app.js file).

Collision detection is done right after all the entities update.

Parameters:
{float} dt
window.requestAnimationFrame interval on the current machine, in seconds.
See:
collisionDetectReact
<inner> updateEntities(dt)
This is called by the update function and loops through all of the objects within your allEnemies array as defined in app.js and calls their update() methods. It will then call the update function for your player object. These update methods should focus purely on updating the data/properties related to the object. Do your drawing in your render methods.
Parameters:
{float} dt
window.requestAnimationFrame interval on the current machine, in seconds.
